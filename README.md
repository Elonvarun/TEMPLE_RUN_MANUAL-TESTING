# Temple Run Mobile Game Testing

## Project Overview

This project demonstrates manual testing of the **Temple Run** mobile
game. The objective is to validate gameplay functionality, user
interface, performance, and overall user experience through structured
manual testing.

The project follows the **Software Testing Life Cycle (STLC)** and
includes professional QA documentation such as test plans, test
scenarios, test cases, and execution reports.

------------------------------------------------------------------------

## Testing Scope

The following modules are covered in this project:

-   Application Launch

-   Gameplay
-   Character Controls
-   Coins and Score
-   Obstacles
-   Power-ups
-   Audio
-   Missions

## Key Observations

-   Gameplay speed increases dynamically as the player progresses.
-   Collision detection works but may fail at sharp turns.
-   Touch gesture responsiveness impacts gameplay performance.
-   Audio feedback enhances user engagement.
